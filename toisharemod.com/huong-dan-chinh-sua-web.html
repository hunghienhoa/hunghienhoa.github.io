Tài liệu quản trị

Hướng dẫn chỉnh sửa website

Website này chạy 100% tĩnh (GitHub Pages). Bạn chỉ cần sửa file JSON và vài đoạn cấu hình ở assets/js/common.js là có thể thay đổi toàn bộ nội dung.

Cấu trúc dữ liệu

Ứng dụng và trò chơi

data/apps.json là nguồn dữ liệu duy nhất cho app/game.

  • Một app = một object trong mảng JSON.
  • Dùng category để phân loại: game hoặc app.

Bài blog

data/posts.json là nguồn dữ liệu cho blog và bài viết.

  • Trang blog đọc danh sách bài.
  • Trang bài viết mở theo URL article.html?id=....

Trò chơi nổi bật

Mục Trò chơi nổi bật (trang chủ) và tab Trò chơi được lấy từ các app có: "category": "game".

Cách chọn game “nổi bật”

Trang chủ đang render mục “Trò chơi nổi bật” từ các app có "featuredGame": true. Bạn chỉ cần bật/tắt cờ này trong data/apps.json. 

{
  "id": "ten-game",
  "category": "game",
  "featuredGame": true
}

Gợi ý: nếu muốn game nào lên trước, hãy đưa object đó lên trên trong file JSON.

Ứng dụng nổi bật

Mục Ứng dụng nổi bật được lấy từ các app có "featuredApp": true và "category": "app". 

{
  "id": "ten-app",
  "category": "app",
  "featuredApp": true
}

Có gì mới

Mục Có gì mới lấy theo cờ "isNew": true trong data/apps.json.

{
  "id": "ten-app",
  "isNew": true,
  "updated": "29/06/2026",
  "version": "1.0.1"
}

Đăng bài blog

Bạn thêm bài mới bằng cách thêm một object vào data/posts.json. Sau đó mở: blog.html để thấy bài mới và article.html?id=slug để xem trang chi tiết.

Mẫu object bài viết

{
  "id": "huong-dan-cai-ipa",
  "title": "Hướng dẫn cài IPA",
  "excerpt": "Mô tả ngắn hiển thị ở trang blog.",
  "date": "29/06/2026",
  "dateIso": "2026-06-29",
  "thumbnail": "assets/img/ten-anh-thumb.jpg",
  "cover": "assets/img/ten-anh-cover.jpg",
  "author": "Admin",
  "readTime": "5 phút đọc",
  "category": "Hướng dẫn",
  "related": ["seo-github-pages"],
  "content": "\\n<p>Nội dung HTML của bài viết.</p>\\n<h2>Mục 1</h2>\\n<p>...</p>\\n"
}

Ảnh thumbnail/cover

  • Chép ảnh vào assets/img/
  • Điền đường dẫn vào thumbnail và cover

Bạn có thể xem thêm file: huong-dan-dang-bai-viet.html nếu cần chi tiết hơn.

Thêm công cụ trên thanh công cụ

Menu công cụ ở topbar đang được cấu hình trong assets/js/common.js (biến toolLinks). Bạn chỉ cần thêm/sửa link trong mảng này. 

const toolLinks = [
  { href: "sign-application.html", label: "iPA Sign" },
  { href: "#", label: "Công cụ 2" }
];

Cách thêm trang công cụ mới

  1. Tạo file mới, ví dụ: tool-xyz.html (đặt ở thư mục gốc).
  2. Nếu có CSS riêng, tạo tool-xyz.css.
  3. Thêm vào toolLinks để hiện trong menu.

Gắn giao diện web chính cho công cụ

Với các trang công cụ (ví dụ sign-application.html), bạn nên giữ đủ giao diện web chính: header (topbar) và thanh điều hướng dưới (mobile nav). Website đang dùng renderShell() để tự render phần này.

Mẫu khung chuẩn nên dùng

<link rel="stylesheet" href="assets/css/styles.css" />
<!-- CSS riêng của công cụ -->
<link rel="stylesheet" href="tool-xyz.css" />

<body data-page="tool">
  <div class="page-shell">
    <header id="site-header"></header>
    <main>... nội dung công cụ ...</main>
    <footer id="site-footer"></footer>
  </div>
  <div id="mobile-nav"></div>
  <script src="assets/js/common.js"></script>
  <script>document.addEventListener("DOMContentLoaded", () => renderShell("tool"));</script>
</body>

Mình đã áp dụng đúng khung này cho sign-application.html để khi mở công cụ vẫn có đủ header và thanh dưới như trang chủ.

Cập nhật sitemap

Nếu bạn muốn SEO tốt hơn, khi thêm trang mới (ví dụ sign-application.html hoặc blog mới) bạn có thể thêm URL vào sitemap.xml.
