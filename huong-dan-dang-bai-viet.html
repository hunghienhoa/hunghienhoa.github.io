Hướng dẫn đăng bài viết lên website

Phần blog của website này đang đọc dữ liệu trực tiếp từ file data/posts.json. Bạn không cần dùng database hay trang quản trị. Chỉ cần thêm ảnh, thêm một object mới vào JSON và đẩy source lên GitHub Pages.

Luồng đăng bài

Website đang hoạt động theo cách rất đơn giản: trang blog.html đọc toàn bộ danh sách bài viết từ data/posts.json, còn trang article.html?id=slug-bai-viet sẽ lấy đúng bài theo trường id để hiển thị.

  1. Chuẩn bị ảnh thumbnail hoặc cover cho bài viết.
  2. Chép ảnh vào thư mục assets/img/.
  3. Mở file data/posts.json.
  4. Thêm một object bài viết mới.
  5. Lưu file, kiểm tra lại dấu phẩy và dấu ngoặc JSON.
  6. Mở thử blog.html và article.html?id=slug.
  7. Nếu muốn SEO sạch hơn, thêm URL bài viết mới vào sitemap.xml.

File cần sửa

File Mục đích
data/posts.json Chứa toàn bộ dữ liệu bài viết để trang blog và trang chi tiết tự render.
assets/img/ Lưu ảnh thumbnail và cover cho bài viết.
sitemap.xml Thêm URL bài mới nếu bạn muốn máy tìm kiếm thấy nội dung nhanh hơn.

Mẫu bài viết mới

Bạn hãy mở data/posts.json, sau đó chèn thêm một object mới theo mẫu dưới đây. Trường content đang dùng HTML chuỗi, vì vậy bạn có thể viết bằng các thẻ như <p>, <h2>, <ul> và <li>. 

{
  "id": "huong-dan-cai-ipa-tren-iphone",
  "title": "Hướng dẫn cài IPA trên iPhone không cần jailbreak",
  "excerpt": "Bài viết giải thích cách sideload file IPA bằng máy tính và những lưu ý quan trọng trước khi cài.",
  "date": "27/06/2026",
  "dateIso": "2026-06-27",
  "thumbnail": "assets/img/huong-dan-cai-ipa-thumb.jpg",
  "cover": "assets/img/huong-dan-cai-ipa-cover.jpg",
  "author": "Tên của bạn",
  "readTime": "6 phút đọc",
  "category": "Hướng dẫn",
  "related": [
    "seo-github-pages",
    "json-content-structure"
  ],
  "content": "\n<p>Đây là đoạn mở đầu của bài viết.</p>\n<p>Bạn có thể viết nhiều đoạn văn liên tiếp để giải thích vấn đề.</p>\n<h2>Chuẩn bị trước khi cài</h2>\n<ul>\n<li>Có file IPA cần cài.</li>\n<li>Có máy tính và cáp kết nối.</li>\n<li>Có công cụ sideload phù hợp.</li>\n</ul>\n<h2>Các bước thực hiện</h2>\n<p>Bạn tiếp tục viết nội dung ở đây.</p>\n"
}

Ý nghĩa từng trường

Trường Cách dùng
id Slug của bài viết. Không nên có dấu cách. Ví dụ: huong-dan-cai-ipa-tren-iphone.
title Tiêu đề chính của bài. Trường này cũng được dùng làm tiêu đề trang chi tiết.
excerpt Mô tả ngắn hiển thị ở danh sách blog và phần meta description.
date Ngày hiển thị ra ngoài, ví dụ 27/06/2026.
dateIso Ngày ở dạng chuẩn, ví dụ 2026-06-27. Nên giữ để dễ mở rộng về sau.
thumbnail Ảnh hiển thị ở trang danh sách bài viết.
cover Ảnh lớn ở đầu trang chi tiết bài viết.
author Tên tác giả hiển thị bên dưới tiêu đề.
readTime Ví dụ 4 phút đọc, 8 phút đọc.
category Nhãn danh mục hiển thị đầu bài viết, ví dụ SEO, Hướng dẫn, Tin tức.
related Mảng chứa id của các bài liên quan đã có sẵn trong posts.json.
content Nội dung HTML của bài viết. Đây là phần quan trọng nhất.

Cách viết nội dung trong trường content

Trường content không phải Markdown mà là HTML nằm trong chuỗi JSON. Vì vậy khi bạn xuống dòng và dùng dấu ngoặc kép, hãy giữ nguyên cú pháp giống các bài đang có sẵn trong data/posts.json.

Bạn nên dùng các thẻ sau:

  • <p> cho đoạn văn
  • <h2> cho tiêu đề phụ
  • <ul> và <li> cho danh sách
  • <code> cho tên file, đường dẫn hoặc lệnh
"content": "\n<p>Đây là đoạn mở đầu.</p>\n<h2>Mục lớn</h2>\n<p>Nội dung tiếp theo ở đây.</p>\n<ul>\n<li>Ý thứ nhất</li>\n<li>Ý thứ hai</li>\n</ul>\n"

Đặt ảnh cho bài viết

Bạn chỉ cần chép ảnh vào thư mục assets/img/, rồi điền đúng đường dẫn vào các trường thumbnail và cover. Nếu muốn dùng chung một ảnh cho cả danh sách và trang chi tiết, bạn có thể đặt cùng một đường dẫn cho hai trường này. 

"thumbnail": "assets/img/ten-bai-viet-thumb.jpg",
"cover": "assets/img/ten-bai-viet-cover.jpg"

Tên file nên viết không dấu, dùng dấu gạch ngang để sau này dễ quản lý hơn.

Mở bài vừa đăng

Sau khi thêm bài vào data/posts.json, bài sẽ tự xuất hiện ở trang blog.html. Trang chi tiết sẽ chạy theo cấu trúc: 

article.html?id=huong-dan-cai-ipa-tren-iphone

Phần id trong URL phải giống hệt trường id bạn đã khai báo trong JSON.

Thêm bài liên quan

Nếu bạn muốn cuối bài có mục bài viết liên quan, hãy điền các id bài đã tồn tại vào trường related. Ví dụ: 

"related": [
  "seo-github-pages",
  "json-content-structure"
]

Nếu bạn điền sai id, bài liên quan sẽ không hiện ra.

Cập nhật sitemap

Website vẫn chạy bình thường nếu bạn chưa sửa sitemap.xml, nhưng nếu muốn SEO tốt hơn thì nên thêm URL bài mới vào file này. 

<url>
  <loc>https://toisharemod.github.io/article.html?id=huong-dan-cai-ipa-tren-iphone</loc>
</url>

Nhớ thay domain mẫu bằng domain GitHub Pages thật của bạn.

Lỗi thường gặp

  • Bài không hiện ở blog: kiểm tra lại dấu phẩy hoặc dấu ngoặc trong data/posts.json.
  • Trang bài viết trắng hoặc lỗi: thường do trường id trên URL không khớp với id trong JSON.
  • Ảnh không hiện: kiểm tra tên file ảnh và đường dẫn trong thumbnail, cover.
  • Bài liên quan không hiện: kiểm tra lại các id trong trường related.
  • Nội dung bị vỡ: kiểm tra xem HTML trong trường content có thiếu thẻ đóng hay không.
